## Symptom

On `/wireframe/calendar`, Shift-dragging blank calendar space no longer leaves a new draft event on the calendar.

## Expected behavior

Shift-drag should create an unsaved draft item in the dragged row/time range and keep the editor open. The follow-up click generated by releasing the drag should not clear that new draft.

## Diagnosis

Two release-time clears were removing the draft:

1. `openDraft()` set the new draft before `resetDraftForm()`, but `resetDraftForm()` also clears the draft. React batched those state updates and left `draft` as `null`.
2. The timeline can emit a deselect event immediately after the Shift-drag release. That deselect should be ignored during the same short release window used for blank-click suppression.

## Plan

1. Add a short suppression window for blank-click clearing after a draft drag finishes.
2. Keep normal blank-click deselect behavior outside that release window.
3. Verify Shift-drag draft creation and the previous blank-click deselect workflow in Playwright.
